# План дипломной работы

1. Проектирование инфраструктуры и сбор данных
    - [ ] Генерация данных: Запуск Synthea для генерации популяции (R4).
    - [ ] Развертывание среды:
        - Настройка HAPI FHIR (R4) с включенным модулем Bulk Export и поддержкой бинарных данных (Binary resources).
    - [ ] Загрузка в CDR:
        - Первичная загрузка бандлов в HAPI FHIR через Batch/Transaction.
    - [ ] Реализация Bulk Export Pipeline:
        - Инициирование операции $export на уровне всей системы (System-level export).
        - Мониторинг статуса (Status Polling) через Content-Location.
        - Выгрузка NDJSON-файлов.
    - [ ] Инъекция в ClickHouse:
        - Создание схем таблиц в ClickHouse.
        - Парсинг NDJSON и загрузка данных.
        - Оптимизация процесса через INSERT FROM SELECT или прямую загрузку файлов.
1. Системный анализ и предобработка данных в ClickHouse
    - [ ] Описание маппинга:
        - Преобразование FHIR-ресурсов (Condition, Observation, Procedure) в аналитический плоский вид.
    - [ ] Анализ полноты данных:
        - Проверка консистентности выгрузки после Bulk Export.
    - [ ] EDA:
        - Статистика по стадиям рака, возрасту и хирургическим вмешательствам.
1. Прогнозирование стадии рака (Classification)
    - [ ] Формирование выборок:
        - Подготовка обучающего набора на основе первичных признаков (симптомы, анализы до операции).
    - [ ] Выбор и обучение модели:
        - Сравнение моделей (RF, XGBoost) для классификации стадий I-II против III-IV.
    - [ ] Валидация:
        - Оценка точности и клинической значимости ошибок.
1. Моделирование выживаемости (Survival Analysis)
    - [ ] Формирование когорты для анализа дожития:
        - Определение моментов постановки диагноза, проведения операции (колэктомии) и конечных точек (ремиссия/смерть).
    - [ ] Метод Каплана-Мейера:
        - Сравнение кривых выживаемости для разных подгрупп.
    - [ ] Регрессия Кокса:
        - Оценка влияния независимых переменных (возраст, коморбидность, стадия) на риск рецидива.
1. Интеграция и архитектура решения (опционально)
    - [ ] Архитектура интеграции:
        - Проектирование пути возврата предиктов модели в FHIR-сервер в виде ресурсов RiskAssessment.
    - [ ] Оценка масштабируемости:
        - Как предложенный Bulk Data Pipeline будет работать на данных миллионов пациентов.
